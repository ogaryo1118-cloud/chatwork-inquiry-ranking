# 📊 お客様の問い合わせランキング自動集計システム

> Chatwork × Claude AI × Google スプレッドシートを連携させた、業務自動化ツール

---

## 📌 概要

社内の Chatwork に届くお客様からの問い合わせを毎日自動で収集・AI解析し、「よくある困りごと」をランキング形式でスプレッドシートに可視化するシステムです。

担当者が手作業でログを確認する必要がなくなり、どの問題が最も多く発生しているかを一目で把握できます。

---

## 🎯 解決する課題

| 課題 | 解決策 |
|------|--------|
| 問い合わせ内容を毎日手作業で確認・集計していた | 毎朝8時に自動収集・集計 |
| 「どの問題が多いか」が感覚でしかわからなかった | AIがキーワードを抽出しランキング表示 |
| 個人情報が外部サービスに渡るリスクがあった | AIに渡す前にGoogle側でPIIをマスク処理 |

---

## 🛠 技術スタック

| 技術 | 用途 |
|------|------|
| **Google Apps Script (GAS)** | バックエンド・自動化の中心 |
| **Chatwork API** | 問い合わせメッセージの自動取得 |
| **Claude API (Anthropic)** | メッセージからキーワードを AI 抽出 |
| **Google スプレッドシート** | データ保存・ランキング可視化 |
| **Gmail API (GAS組み込み)** | 日次・週次・月次レポートの自動送信 |

---

## ✨ 主な機能

- **毎日自動収集** — 設定したChatworkルームから本日のメッセージを自動取得
- **AI キーワード抽出** — Claude APIがメッセージを分析し「困りごと」を分類
- **ランキング表示** — 本日 / 今週 / 今月 の3期間でランキングをワンクリック表示
- **自動レポート送信** — 管理者メールアドレスへ日次・週次・月次レポートを自動送信
- **設定シート保護** — パスワード認証で設定シートを誤操作から保護

---

## 🔒 セキュリティ設計

```
Chatwork → Google Apps Script → [PIIマスク処理] → Claude API → キーワードのみ返却
```

**個人情報はClaude APIに渡りません。**

Chatworkから取得したメッセージは、Claude APIへ送信する前に Google サーバー上で以下を自動マスクします：

- メールアドレス → `[メールアドレス]`
- 電話番号 → `[電話番号]`
- URL → `[URL]`
- 郵便番号 → `[郵便番号]`
- 敬称付き名前（田中さん、山田様など）→ `[お名前]`

## 📂 ファイル構成

```
/
├── Code.gs              # メインスクリプト（GAS）
│   ├── setup()          # 初回セットアップ
│   ├── fetchAndProcessMessages()  # メッセージ取得・AI解析
│   ├── _sanitizePersonalInfo()    # 個人情報マスク処理
│   ├── _callClaudeAPI()           # Claude API連携
│   ├── setupDashboardButtons()    # ダッシュボードUI構築
│   └── setupSettingsProtection()  # 設定シート保護
└── dashboard.html       # ウェブアプリUI（GAS HTML Service）
```

---

## 🚀 セットアップ手順

1. 新しい Google スプレッドシートを作成する
2. 拡張機能 → Apps Script を開く
3. `Code.gs` の内容を貼り付ける
4. ＋ボタンで `dashboard.html` を追加し内容を貼り付ける
5. `setup` 関数を実行してシートとトリガーを初期化する
6. シート「設定」に以下を入力する
   - Chatwork APIキー（Settings > API Token）
   - Claude APIキー（Anthropic Console）
   - 管理者メールアドレス
7. デプロイ → ウェブアプリとして公開しURLを社内共有

---

## 💡 工夫したポイント

- **Claude へのプロンプト設計** — キーワード抽出の精度を高めるため、出力形式をJSON固定にしてパースエラーを防止
- **PII マスキングの二重防御** — GASでの正規表現マスク＋Claudeプロンプトでの明示的な除外指示
- **チェックボックス式UI** — GASの `onEdit` トリガーを活用し、ボタンのような操作感を実現
- **レート制限対策** — Chatwork API / Claude API それぞれにウェイト処理を実装
